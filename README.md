# SC Tickets

**SC Tickets** — это русскоязычное Flask-приложение для управления событиями, продажи и проверки QR-билетов.

Приложение включает:
- регистрацию пользователей и подтверждение электронной почты;
- создание и редактирование мероприятий в админ-панели;
- покупку билетов с проверкой вместимости;
- генерацию PDF-билетов с QR-кодом;
- личный кабинет пользователя и страницу профиля;
- входной контроль и отметку билетов как использованных.

---

## Основные возможности

- Пользовательская регистрация с email-подтверждением.
- Авторизация и персональный кабинет с обзором билетов.
- Список событий с показом доступных мест и процента продаж.
- Покупка билетов и генерация PDF-файла.
- Админ-панель `/admin` для управления пользователями, событиями и билетами.
- Сканирование QR-кода через `/admin/scan` для отметки билета как использованного.
- Статистика по пользователям, событиям и использованным билетам.

---

## Структура проекта

- `server.py` — точка запуска Flask-приложения, регистрация blueprint-ов и настройки сессий.
- `main.py` — генерация PDF-билетов и QR-кодов.
- `auth.py` — регистрация, вход, выход и подтверждение email.
- `events.py` — публичная афиша, покупка билетов, кабинет и профиль.
- `admin.py` — админские маршруты для управления мероприятиями, пользователями и билетами.
- `database.py` — SQLite-слой, модели пользователей, событий и билетов.
- `mail.py` — отправка кода подтверждения по SMTP.
- `templates/` — HTML-шаблоны для пользовательского интерфейса и админки.

## Быстрый запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

Откройте файл `.env` и заполните параметры:

- `SECRET_KEY` — секретный ключ Flask для сессий;
- `MAIL_FROM` — адрес отправителя для подтверждения email;
- `MAIL_PASSWORD` — пароль или app-password SMTP;
- `FLASK_DEBUG` — `1` для локальной отладки;
- `ENABLE_NGROK` — `1`, если хотите открыть публичный адрес через ngrok;
- `SESSION_COOKIE_SECURE` — `1` при HTTPS;
- `DB_PATH` — необязательный путь к SQLite-базе данных;
- `VERIFY_TOKEN_TTL_MINUTES` — срок действия кода подтверждения.

Запустить сервер:

```powershell
python server.py
```

По умолчанию приложение будет доступно на `http://127.0.0.1:5000`.

## Администрирование

1. Зарегистрируйте пользователя через веб-интерфейс.
2. Сделайте его администратором командой:

```powershell
python make_admin.py user@example.com
```

3. Войдите в систему и перейдите в `/admin`.

Доступные разделы:
- `Dashboard` — общая статистика по пользователям, событиям и билетам;
- `Events` — создание, редактирование и удаление мероприятий;
- `Users` — список пользователей и изменение их роли;
- `Tickets` — групповая статистика билетов;
- `Scan` — сканирование QR-билетов.

## Как работает билеты и проверка

- При покупке билета создаётся уникальный `ticket_id`.
- PDF-билет генерируется в `Generated_tickets/`.
- Если в проекте есть файл `ghb.pdf`, он используется как шаблон для наложения QR-виджета.
- Если шаблона нет — PDF создаётся напрямую с данными события.
- Проверка на входе `/used` помечает билет как использованный и блокирует повторное применение.

## Полезные заметки

- Сгенерированные PDF-файлы сохраняются в `Generated_tickets/`.
- Рекомендуется добавить `tickets.db`, `__pycache__/`, `Generated_tickets/`, `.env` и другие артефакты в `.gitignore`.
- Почтовая отправка работает через Gmail SMTP (`smtp.gmail.com:465`); для аккаунтов Gmail может потребоваться пароль приложения.
- В админке можно редактировать цветовую схему карточек события: фон, акцент и цвет текста.
